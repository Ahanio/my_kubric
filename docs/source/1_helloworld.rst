Helloworld
==========

The typical Kubric workflow involves a worker file, that describes the scene and how to render it.
Let us look at the very simple `examples/helloworld.py <https://github.com/google-research/kubric/blob/main/examples/helloworld.py>`_

First, we need to create a (default) scene and attach the renderer (Blender) to it.

.. literalinclude:: /../examples/helloworld.py
  :lineno-start: 21
  :lines: 21-22
  :linenos:

Next we will create a floor (i.e. cube), and add a sphere on top of it (with default materials). We also add a directional light to illuminate the scene, and a camera for rendering:

.. literalinclude:: /../examples/helloworld.py
  :lineno-start: 25
  :lines: 25-28
  :linenos:

We can ask Blender to render an image from the given camera:

.. literalinclude:: /../examples/helloworld.py
  :lineno-start: 31
  :lines: 31
  :linenos:

  renderer.render_still("helloworld.png")

.. image:: /images/helloworld.png
   :width: 200pt
   :alt: Image rendered by Kubric (via Blender) 
   :align: center
   
Or ask Kubric to generate the blender file representing this scene (you will be able to open the ``blend`` file in Blender and inspect it as visualized in the screenshot below):

.. literalinclude:: /../examples/helloworld.py
  :lineno-start: 32
  :lines: 32
  :linenos:

  renderer.save_state("helloworld.blend")
